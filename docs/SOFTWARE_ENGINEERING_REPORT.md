## ۱. مقدمه و اهداف

پروژه تات‌زیور یک فروشگاه آنلاین زیورآلات زنانه دست‌ساز است که با چارچوب Django و معماری API-محور پیاده‌سازی شده است. هدف از این مستند، ارائهٔ یک گزارش جامع مهندسی نرم‌افزار برای نسخه ۱ بک‌اند این پروژه می‌باشد.

### اهداف نسخه ۱:
- ایجاد یک بک‌اند کامل با ۱۶ اپلیکیشن تخصصی
- پشتیبانی از فروشگاه فیزیکی با احراز هویت JWT و OTP
- مدیریت موجودی، سبد خرید، پرداخت و رهگیری مرسوله
- هوش مصنوعی آفلاین (چت‌بات و استایل‌یاب) با Ollama
- حسابداری خودکار، CRM، بازاریابی و چرخه عمر مشتری
- زیرساخت چندزبانه و قابلیت اشتراک‌گذاری اجتماعی
- تست‌های کامل (۶۱ تست پاس‌شده) و مستندات Swagger

## ۲. اصول پیمان‌نامه همکاری

۱. **تحویل کامل فایل** – هر فایل بدون تکه‌تکه ارائه می‌شود.  
۲. **مسیرهای مطلق از ریشه** – تمام ارجاعات از مسیر `D:\py\TatZivar` محاسبه می‌شود.  
۳. **اولویت آفلاین** – بدون CDN، کتابخانه‌ها محلی، سرویس‌های Mock برای توسعه.  
۴. **جداسازی منطق** – منطق کسب‌وکار در `services.py`، ویوها فقط درخواست و پاسخ.  
۵. **مستندسازی علّی** – هر تصمیم با «چرا» همراه است.  
۶. **اسکریپت‌محوری** – اسکریپت‌های Idempotent و خودمدیریت خطا برای تمام عملیات.

## ۳. معماری کلی سیستم

بک‌اند بر اساس معماری لایه‌ای (Layered Architecture) طراحی شده است:

- **لایهٔ نمایش (API):** ویوهای Django REST Framework (کلاس‌های مبتنی بر APIView و Generic Views).
- **لایهٔ منطق (Services):** فایل‌های `services.py` در هر اپلیکیشن مسئول تمام محاسبات و تعامل با مدل‌ها هستند.
- **لایهٔ داده (Models):** مدل‌های Django برای تعریف ساختار دیتابیس.

ارتباط با فرانت‌اند از طریق REST API (JSON) و احراز هویت JWT انجام می‌شود. مدل‌های زبانی محلی (Ollama) به‌صورت مستقیم از سرویس‌های مربوطه فراخوانی می‌شوند.

## ۵. اپلیکیشن‌ها و وظایف

### ۵.۱ products
مدیریت محصولات، دسته‌بندی‌ها، تصاویر.  
**مدل‌ها:** Category, Product, ProductImage, ProductType, MaterialType, StyleType

**مدل‌ها:** Cart, CartItem, Order, OrderItem  
**سرویس‌ها:** `calculate_cart_total`, `create_order_from_cart`, `update_stock`

### ۵.۳ users
احراز هویت، پروفایل، آدرس‌ها، علاقه‌مندی‌ها.  
**مدل‌ها:** Profile, Address, Wishlist, WishlistItem, PhoneOTP

### ۵.۴ blog

**مدل‌ها:** ArticleCategory, Article, CustomerGallery

### ۵.۵ customize
سفارش شخصی‌سازی زیورآلات.  
**مدل:** CustomOrder با فیلدهای color, initials, birthstone

### ۵.۶ social

**مدل‌ها:** CustomerSegment, ProductAnalytics, MarketingCampaign, DailyStats

**مدل‌ها:** ChatSession, ChatMessage, ChatRule

لاگ تغییرات موجودی و هشدار کمبود.

### ۵.۱۴ translations

### ۵.۱۵ accounting

## ۶. مدل‌های داده

در ادامه، مدل‌های اصلی و روابط بین آن‌ها تشریح می‌شود:

- **Product ← Category (ForeignKey)** – هر محصول به یک دسته تعلق دارد.
- **ProductImage ← Product (ForeignKey)** – تصاویر چندگانه برای هر محصول.
- **Cart ← User (ForeignKey, nullable) / session_key** – سبد خرید کاربر یا مهمان.
- **CartItem ← Cart, Product (ForeignKey)** – آیتم‌های سبد خرید.
- **Order ← User (ForeignKey)** – سفارش نهایی.
- **OrderItem ← Order, Product (ForeignKey)** – آیتم‌های سفارش.
- **Profile ↔ User (OneToOne)** – پروفایل کاربر.
- **Wishlist ↔ User (OneToOne)** – علاقه‌مندی‌ها.
- **PhoneOTP ← phone (CharField)** – کد تأیید موقت.
- **PaymentTransaction ← Order, PaymentMethod (ForeignKey)** – تراکنش پرداخت.
- **FraudCheck ← PaymentTransaction (OneToOne)** – بررسی تقلب.
- **Invoice ← Order (OneToOne)** – فاکتور فروش.
- **Shipment ← Order, ShippingCarrier (ForeignKey)** – مرسوله.
- **LifecycleTask ← Order (ForeignKey)** – وظایف چرخه عمر.

## ۷. احراز هویت و امنیت

دو روش احراز هویت وجود دارد:

- **JWT (SimpleJWT):** ثبت‌نام با نام‌کاربری/ایمیل و رمز عبور، دریافت access/refresh token.
- **OTP:** درخواست کد به شماره موبایل (Mock SMS) و تأیید آن برای ورود/عضویت.

### امنیت:
- تمام APIهای حساس نیازمند احراز هویت هستند.
- تنظیمات CORS برای دسترسی موبایل و فرانت‌اند فعال است.
- سیستم کشف تقلب در اپ `payment` تعداد تراکنش‌های بالا از یک IP و تکرار دستگاه را بررسی می‌کند.

## ۹. یکپارچگی و جریان‌های اصلی

### جریان خرید:

۱. کاربر محصول را به سبد اضافه می‌کند (`POST /api/cart/add/`).  
۲. کاربر با JWT یا OTP احراز هویت می‌شود و آدرس را ثبت می‌کند.  
۳. با `POST /api/checkout/` سفارش ایجاد می‌شود و موجودی کاهش می‌یابد.  
۴. کاربر روش پرداخت را انتخاب کرده و درخواست پرداخت می‌دهد (`POST /api/payment/initiate/`).  
۵. پس از تأیید پرداخت (Mock)، تراکنش مالی ثبت و فاکتور تولید می‌شود.  
۶. ادمین مرسوله را ایجاد و کد رهگیری را ثبت می‌کند (`POST /api/admin/shipments/create/`).  
۷. وظایف چرخه عمر (نظرسنجی) زمان‌بندی می‌شود.  
۸. کاربر می‌تواند از طریق `/api/track/` مرسوله را پیگیری کند.

## ۱۰. نصب و راه‌اندازی

مراحل نصب روی سیستم ویندوز:

۱. **ایجاد محیط مجازی:**  
   ```bash
   python -m venv .venv
   .venv\Scripts\activate
   ```

۲. **نصب وابستگی‌ها:**  
   ```bash
   pip install -r requirements.txt
   ```

۳. **اعمال مهاجرت‌ها:**  
   ```bash
   python manage.py migrate
   ```

۴. **(اختیاری) بارگذاری داده‌های نمونه:**  
   ```bash
   python scripts/seed_data.py
   ```

۵. **اجرای سرور:**  
   ```bash
   python manage.py runserver
   ```

۶. **دسترسی به پنل ادمین:**  
   - آدرس: `http://127.0.0.1:8000/admin/`
   - اگر seed اجرا شده: `admin` / `admin123`

۷. **مستندات Swagger:**  
   - آدرس: `http://127.0.0.1:8000/api/docs/`

## ۱۴. جمع‌بندی و وضعیت نهایی
